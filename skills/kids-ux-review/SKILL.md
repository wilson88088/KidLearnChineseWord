# 兒童教育 App UX 審查技能

## 目的
以專業 UX 設計師的角度，審查兒童教育類 App 的介面設計，確保對 6-12 歲兒童具有吸引力和易用性。

## 審查維度

### 1. 視覺吸引力（Visual Appeal）
檢查項目：
- 色彩是否明亮活潑且對比足夠？
- 是否有吉祥物或角色增加情感連結？
- 動畫和轉場是否流暢有趣？
- 圖示是否直覺易懂？
- 字體大小是否適合兒童閱讀（至少 18px）？

評分標準：1-5 分，3 分為及格

### 2. 觸控友善度（Touch Friendliness）
檢查項目：
- 觸控目標至少 44x44px
- 按鈕間距至少 8px
- 手寫區域是否夠大（建議至少 280x280px）
- 滑動操作是否流暢
- 是否支援手寫筆壓力感應

### 3. 導航清晰度（Navigation Clarity）
檢查項目：
- 兒童能否不需大人幫助就能操作？
- 返回鍵是否明顯？
- 目前位置是否清楚？
- 功能入口是否一目了然？
- 是否有新手引導？

### 4. 遊戲化元素（Gamification）
必要元素檢查：
- [ ] 成就系統 / 徽章
- [ ] 連擊計數
- [ ] 每日簽到
- [ ] 進度視覺化
- [ ] 驚喜獎勵
- [ ] 音效回饋
- [ ] 動畫慶祝效果

### 5. 正向回饋機制（Positive Feedback）
檢查項目：
- 答對時是否有即時獎勵感？
- 答錯時是否鼓勵而非懲罰？
- 進步是否被可視化？
- 音效是否令人愉悅？

### 6. 無障礙設計（Accessibility）
- WCAG AA 對比度標準
- 語音輔助支援
- 大字體模式
- 色盲友善色彩

## 兒童心理設計原則

1. **即時滿足**：每次操作都要有反饋（0.3秒內）
2. **漸進挑戰**：難度循序漸進，避免挫折
3. **安全探索**：錯誤不會造成不可逆結果
4. **社交認同**：可分享成就，獲得肯定
5. **自主控制**：讓孩子感覺自己在掌控
