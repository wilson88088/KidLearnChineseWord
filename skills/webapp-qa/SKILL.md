# Web App 品質測試技能

## 目的
對 HTML Web App 進行系統化的品質測試，找出 Bug、效能瓶頸、安全漏洞和可用性問題。

## 測試流程

### 第一步：靜態分析
讀取原始碼，檢查：

1. **JavaScript 語法**
   - 使用 Node.js `new Function()` 驗證語法正確性
   - 檢查未定義的變數引用
   - 檢查事件綁定是否完整

2. **HTML 結構**
   - 標籤配對是否正確
   - ID 是否唯一
   - 表單元素是否有適當 label

3. **CSS 檢查**
   - 未使用的 CSS 規則
   - 媒體查詢是否覆蓋常見裝置
   - z-index 衝突

### 第二步：流程測試
追蹤每個使用者流程：

```
流程模板：
1. 前置條件：[描述起始狀態]
2. 操作步驟：[逐步操作]
3. 預期結果：[應該看到什麼]
4. 實際結果：[程式碼分析後的結果]
5. 狀態：✅ 通過 / ❌ 失敗 / ⚠️ 有疑慮
```

### 第三步：邊界情況測試
必查項目：
- 空資料狀態（首次使用）
- 超長文字輸入
- 特殊字元輸入（引號、HTML 標籤、emoji）
- 快速連續點擊
- localStorage 已滿的情況
- 外部 CDN 載入失敗
- 瀏覽器 API 不支援（Speech、Canvas）

### 第四步：安全性檢查
- XSS 漏洞（用戶輸入未轉義）
- innerHTML 中的動態內容
- eval() 或 Function() 的使用
- 外部腳本的完整性（SRI）
- localStorage 中的敏感資料

### 第五步：效能檢查
- DOM 操作頻率
- 事件監聽器的管理（記憶體洩漏）
- 大型資料的序列化/反序列化
- 圖片和 base64 的大小控制
- 動畫的 GPU 加速

```markdown
| 流程 | 結果 | 備註 |
|------|------|------|

```

## 自動修復模式

當使用者要求自動修復時：
1. 優先修復 Critical 和 High 級別的問題
2. 每次修改後重新驗證不會引入新問題
3. 使用 Edit 工具進行精確修改
4. 修改後列出所有變更的摘要
